GNS3 3.1 使用哪些端口？如何修改端口范围？​

用途默认端口范围说明
主服务端口80 / 443 / 3080GNS3 VM 默认端口
Console 端口5000-10000设备控制台连接
VNC 端口5900-10000VNC 连接
UDP 通信端口10000-30000设备间链路通信

所有端口范围可在 gns3_server.conf 配置文件中自定义修改。

计算节点连接时报 401 Unauthorized 错误？​

  1. 检查计算节点配置中的 compute_username 和 compute_password 是否与控制器端传递的凭证匹配
  2. 验证计算节点的配置文件是否正确加载
  3. 确保使用 [Server] 部分（而非 [Controller]）

如果计算节点未设置 compute_password，启动时会自动生成随机的 16 位字符密码，导致控制器无法认证。

计算节点报 "No common subnet" 错误？​

主要原因： 控制器的 host 设置为 0.0.0.0，导致控制器注册为 127.0.0.1，计算节点无法访问。

解决方法： 将控制器的 host 设置为其实际的 LAN IP 地址（如 host = 192.168.1.104），而非 0.0.0.0。如果使用 DHCP，建议在路由器上设置静态租约或使用 mDNS（.local 域名）。
